2. Browser Functionality & Web Navigation

Swift Go Browser is a full-featured web browser that allows you to access and navigate websites across the internet. The browser uses WebView technology (specifically flutter_inappwebview) to render web pages with full JavaScript support, media playback capabilities, and modern web standards compliance. When you browse websites, the app loads and displays web content just like any standard browser. The browser supports multiple tabs, allowing you to have several web pages open simultaneously. It includes a smart address bar that accepts both URLs and search queries, automatically detecting whether you've entered a website address or a search term. The browser maintains a navigation history for each tab, enabling you to go back and forward through previously visited pages. We support both HTTP and HTTPS protocols, with visual indicators to show you when you're on a secure connection. The browser can handle various web technologies including forms, cookies, local storage, and multimedia content. We also support deep linking, allowing you to open web links from other applications directly in Swift Go Browser. The browser is optimized for performance with hardware acceleration enabled for smooth scrolling and fast page rendering.

3. Permissions System & Runtime Requests

Swift Go Browser implements a privacy-focused permission system that requests access to device features only when they are actually needed by a website or app feature. We NEVER request permissions at app startup or installation. All permissions are requested at runtime when a specific feature requires them. Here's how each permission works:

Internet Permission (INTERNET): This permission is automatically granted and allows the app to access the internet for browsing websites. This is essential for browser functionality and does not require user approval.

Storage Permissions (READ_EXTERNAL_STORAGE, WRITE_EXTERNAL_STORAGE): These permissions are requested only when you initiate a download or when a website needs to access files. On Android 11 and above, we use scoped storage which provides better privacy protection. We never access your files without your explicit action.

Camera Permission: Requested only when a website asks to use your camera (e.g., for video calls, taking photos on web apps). You'll see a permission dialog explaining why the camera is needed before any access is granted.

Microphone Permission: Requested only when a website needs audio input (e.g., voice calls, voice search on websites). The permission dialog clearly states which feature needs microphone access.

Location Permission (ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION): Requested only when a website asks for your location (e.g., maps, weather services). You have full control to allow or deny location access, and you can revoke it anytime from Android settings.

Notification Permission (POST_NOTIFICATIONS): On Android 13+, this permission is NEVER automatically requested by our app. Notifications are only shown if you manually enable them from Android system settings. We respect your choice to keep notifications disabled.

Phone Permission: Requested only when a website needs to make phone calls or access phone state. This is rare and only happens with specific web applications.

All permission requests include a clear explanation of why the permission is needed and which feature requires it. You can deny any permission, and the app will continue to work, though the specific feature requiring that permission may not function. You can review and change all permissions anytime from your device's Settings > Apps > Swift Go Browser > Permissions.

4. Security System & HTTPS Detection

Swift Go Browser includes a comprehensive security system designed to protect you while browsing. The browser actively monitors the security status of every website you visit and provides clear visual indicators about connection security. When you visit a website, our security system checks whether the connection uses HTTPS (secure) or HTTP (unsecure) protocol. For HTTPS websites, you'll see a lock icon or secure indicator in the address bar, confirming that your connection is encrypted and protected from eavesdropping. For HTTP websites, you'll see a warning indicator showing that the connection is not secure, and any data you enter could potentially be intercepted.

Our security system also handles SSL/TLS certificates, validating that websites are who they claim to be. While we do allow users to proceed to websites with certificate issues (to maintain browsing flexibility), we clearly warn you about potential security risks. The browser supports modern encryption standards and automatically upgrades connections to HTTPS when available. We also implement protection against mixed content, warning you when a secure page tries to load insecure resources.

Additionally, our security system includes protection against malicious downloads and suspicious file types. When you attempt to download files, the system checks file extensions and warns you about potentially dangerous file types like APK files or executables. The browser maintains a secure sandbox environment, isolating web content from your device's core system to prevent malicious websites from accessing sensitive device features without permission. We regularly update our security protocols to protect against emerging threats and vulnerabilities.

6. Download System & File Management

Swift Go Browser includes a powerful download manager that allows you to download files from websites directly to your device. The download system is built using flutter_downloader library and provides comprehensive download management capabilities. Here's how the download system works and what permissions it requires:

When you click a download link or a file on a website, the browser automatically detects common file types (including MP4, MP3, PDF, ZIP, JPG, PNG, APK, DOC, XLS, PPT, and many others) and initiates the download process. Before starting any download, the app requests storage permission if not already granted. On Android 11 and above, we use scoped storage which means we only access the Downloads folder, not your entire device storage.

The download manager provides real-time progress tracking, showing you the download speed, percentage completed, and estimated time remaining. All downloads run in the background, allowing you to continue browsing while files download. You can pause, resume, or cancel downloads at any time from the downloads screen. The system supports multiple simultaneous downloads, managing them efficiently to optimize your device's resources.

Each download is assigned a unique notification that shows progress in your notification panel. When a download completes, you receive a completion notification with options to open the file or view it in your file manager. Failed downloads are clearly marked with error messages, and you can retry them with a single tap. The download history is maintained within the app, showing all completed, failed, and cancelled downloads.

Downloaded files are stored in your device's standard Downloads folder (typically /storage/emulated/0/Download/) with proper file naming to prevent conflicts. We use Android's FileProvider system to securely share downloaded files with other apps when you choose to open them. The download system respects your device's storage limits and will notify you if there's insufficient space before starting a download.

Permissions Used: READ_EXTERNAL_STORAGE (to check existing files), WRITE_EXTERNAL_STORAGE (to save downloaded files on Android 10 and below), DOWNLOAD_WITHOUT_NOTIFICATION (to manage downloads), and INTERNET (to download files from the web).

8. Data Storage (Bookmarks, History, Cache & Cookies)

Swift Go Browser stores various types of data locally on your device to enhance your browsing experience and provide essential browser functionality. All data is stored locally on your device and is NEVER transmitted to our servers or any third-party services (except as required by websites you visit). Here's a detailed explanation of what data we store and how it's managed:

Browsing History: The app maintains a record of websites you visit, including the page title, URL, and timestamp. This history is stored locally using the Hive database and Provider state management. History allows you to quickly revisit previously viewed pages and provides search suggestions. You can view your complete history from the History screen, search through it, and delete individual entries or clear all history at once. History data never leaves your device.

Bookmarks: When you bookmark a webpage, we store the page title, URL, favicon (website icon), and the date you bookmarked it. Bookmarks are stored locally using Provider state management and persist across app sessions. You can organize, edit, and delete bookmarks anytime from the Bookmarks screen. Bookmark data is private and stored only on your device.

Cookies: Websites you visit may store cookies in the browser to remember your login sessions, preferences, and other site-specific data. These cookies are managed by the WebView component and are stored in the app's private data directory. Cookies are essential for many websites to function properly (e.g., keeping you logged in). You can clear all cookies from the "Clear Browser Data" screen in app settings.

Cache: The browser caches website resources (images, scripts, stylesheets) to speed up page loading on repeat visits. Cached data is stored in the app's cache directory and is managed automatically by the WebView engine. Cache helps reduce data usage and improve browsing speed. You can clear the cache anytime from the "Clear Browser Data" screen.

Local Storage & Session Storage: Websites can store data locally using HTML5 Local Storage and Session Storage APIs. This data is managed by the WebView and is isolated per website. It's used by websites for various purposes like saving preferences, offline functionality, and app state. You can clear this data from the "Clear Browser Data" screen.

App Preferences: We store your app settings and preferences using SharedPreferences, including your selected search engine, dark mode preference, whether you've seen the onboarding screens, and other app configuration settings. This data is minimal and only used to customize your app experience.

Download Records: Information about your downloads (file name, URL, download date, file size, file path) is stored locally to maintain your download history. This allows you to track completed downloads and access downloaded files easily.

Clear Browser Data: You have complete control over your stored data. The app includes a "Clear Browser Data" feature that allows you to selectively delete browsing history, cookies, cache, and other stored data. You can choose what to delete and clear it with a single tap.

Data Location: All data is stored in the app's private directory on your device (/data/data/com.abhi.swiftgo/ and /storage/emulated/0/Android/data/com.abhi.swiftgo/). This data is protected by Android's app sandbox and is not accessible to other apps. When you uninstall Swift Go Browser, all stored data is automatically deleted by Android.

9. Incognito Mode (Private Browsing)

Swift Go Browser includes an Incognito Mode feature that provides enhanced privacy for your browsing sessions. When you browse in Incognito Mode, the app modifies its data storage behavior to ensure your browsing activity remains private. Here's exactly how Incognito Mode works and what it does:

What Incognito Mode Does:

  • No History Saved: Websites you visit in Incognito Mode are NOT added to your browsing history. Once you close the incognito tab, there's no local record of the websites you visited.
  • No Cookies Persisted: Cookies created during an incognito session are temporary and are deleted when you close the incognito tab. Websites won't remember you after the session ends.
  • No Cache Retention: Cached resources from incognito browsing are not permanently stored. The cache is cleared when you close the incognito session.
  • Separate Session: Incognito tabs run in a separate browsing context from your normal tabs. This means you can be logged into a website in a normal tab and browse the same website logged out (or with a different account) in an incognito tab.
  • No Autofill: Form data and search queries entered in Incognito Mode are not saved for autofill suggestions.

What Incognito Mode Does NOT Do:

  • Not Anonymous: Incognito Mode does NOT make you anonymous on the internet. Websites you visit can still see your IP address, and your internet service provider (ISP) can still see your browsing activity.
  • No Download Privacy: Files you download in Incognito Mode are still saved to your device's Downloads folder and appear in your download history.
  • No Bookmark Privacy: If you bookmark a page in Incognito Mode, that bookmark is saved permanently like any other bookmark.
  • No Network Privacy: Your network administrator, ISP, or the websites you visit can still track your activity. Incognito Mode only affects local data storage on your device.

How to Use Incognito Mode: You can open an incognito tab from the tab switcher or menu. Incognito tabs are visually distinguished from normal tabs (typically with a different color scheme or icon). You can have both normal and incognito tabs open simultaneously. When you close all incognito tabs or exit the app, all incognito session data is automatically cleared.

Use Cases: Incognito Mode is useful when you want to browse without leaving traces on your device, such as when shopping for gifts, using a shared device, or accessing websites without affecting your browsing history and cookies.

12. Data Security & Protection

We take the security of your data seriously and implement appropriate technical and organizational measures to protect your information from unauthorized access, alteration, disclosure, or destruction. Here's how we protect your data:

Local Storage Security: All data stored by Swift Go Browser (history, bookmarks, preferences, download records) is stored in the app's private directory, which is protected by Android's app sandbox. Other apps cannot access this data without root access to your device. The app uses secure storage mechanisms provided by Android, including SharedPreferences for settings and Hive database for structured data.

Encryption in Transit: When you visit HTTPS websites, your connection is encrypted using SSL/TLS protocols. This encryption is handled by the website and the Android WebView component. We display security indicators to help you identify secure connections. However, we cannot guarantee the security of HTTP (non-encrypted) connections, which is why we warn you about insecure websites.

No Remote Storage: We do NOT store your browsing data, history, bookmarks, or any personal information on remote servers. Everything stays on your device. This significantly reduces the risk of data breaches, as there's no central database of user information that could be compromised.

Secure Permissions: We implement runtime permission requests, ensuring that sensitive device features (camera, microphone, location, storage) are only accessed when explicitly needed and authorized by you. Permissions are enforced by the Android operating system.

Third-Party Security: For third-party services (AdMob, Firebase), we rely on Google's security infrastructure, which includes encryption, access controls, and regular security audits. Google is a leader in data security and complies with international security standards.

Download Security: Downloaded files are scanned by your device's security software (if installed). We warn you about potentially dangerous file types (APK, EXE) before downloading. However, we recommend using antivirus software and being cautious about what files you download from the internet.

App Updates: We regularly update the app to patch security vulnerabilities, update dependencies, and improve security features. We recommend keeping Swift Go Browser updated to the latest version for optimal security.

Limitations: While we implement strong security measures, no system is completely secure. You are responsible for:

  • Keeping your device secure with a lock screen password/PIN.
  • Not rooting your device, which can compromise Android's security model.
  • Being cautious about which websites you visit and what information you provide to them.
  • Keeping your device's operating system and security patches up to date.
  • Using secure Wi-Fi networks and avoiding public Wi-Fi for sensitive transactions.

Data Breach Notification: In the unlikely event of a data breach affecting user data, we will notify affected users in accordance with applicable laws and regulations. However, since we don't store user data on our servers, the risk of such a breach is minimal.
